## P0（第11回）: 1ターミナル → web → 音

### AIへの依頼（要旨）
- テーマ: 複数AIの同時運用を音で伝える
- 最終目標: VOICEVOXで「〇〇版が承認待ち/完了です」と読み上げ ＋ Discord通知

- 制約（軽量重視）: サーバはNode標準機能のみ（npm install不要）

- `curl` でイベントを送ると音が鳴る構成

### 実施済み（2026-07-01）: 複数AIダッシュボード

**設計判断**：サーバを「現在状態の持ち主」に変更した。
それまでのサーバはイベントを素通しするだけだったので、ブラウザを後から開くと何も表示されなかった。
サーバが各AIの最新状態を `Map` で保持し、SSE接続時にスナップショットを送るようにした結果、
「あとから画面を開いても現在の盤面が見える」ダッシュボードとして成立した。

- 同じ `ai` 名は上書き（重複して増えない）
- 要対応（承認待ち→エラー）を上に、同順位なら新しい順にソート
- 経過時間表示（1秒ごとに更新）、`/clear` で一覧リセット

**動作確認**：4AIを別状態で登録 → 新規タブで開いてスナップショットが届くこと、
既存AIの状態変更が重複せず並び替わること、新規AIが要対応で最上段に割り込むこと、
クリアで盤面のみ消えてログが残ることを、ブラウザ実機で確認。

### 実施済み（2026-07-14）: セッション追跡・実行中表示・音トグル・永続化

実運用レビューで挙がった問題への対応。

**設計判断**
- 同一プロジェクトの複数セッションが盤面上で1つに潰れる問題 → AI名を「プロジェクト名#セッションID先頭6文字」に変更（hookがstdinのJSONから `session_id` を取得）
- セッション終了の残骸が盤面に溜まる問題 → SessionEnd hook が `state=ended` を送り、サーバが盤面から自動削除
- 「今どのAIが動いているか」が見えない問題 → UserPromptSubmit hook で `state=working` を送信
- 完了音が毎ターン鳴る通知疲れ → 状態ごとの音ON/OFFトグル（localStorageに保存。試聴はOFFでも鳴る）
- サーバ再起動で盤面消失 → `.state.json` へ永続化し起動時に復元
- エラーの自動検出（PostToolUseFailure hook）は見送り。ツール失敗はClaudeが自力でリカバリする通常動作の一部で、毎回⛔＋ブザーでは誤報だらけになるため

**動作確認**
- pipe-test: 偽session_idで4 hook全部の発火・盤面反映・ended削除を確認
- 実機: 同一ディレクトリで claude -p を2本並行実行 → `ai-sound-monitor#802488` と `#925f1e` が別行で🔄実行中→✅完了→終了時に削除、のライフサイクルを確認
- サーバ再起動 → 盤面2件が復元されることを確認

**サブエージェント運用の学び**
- 通常ファイル(css/html/gitignore)の編集はサブエージェントで問題なし
- `.claude/settings.json`（hook設定）への書き込みはサブエージェントには許可されず、2エージェント連続で停止 → hook設定だけはメインセッションで実施
